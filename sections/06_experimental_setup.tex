\chapter{Experimental Setup}

\section{Physical Cluster}
The project uses three physical MacBook machines connected through a local network. The benchmark mode uses CPU execution to match the course emphasis on MPI processes and processor assignment. GPU/MPS execution is kept for demonstration and live-camera use rather than formal CPU speedup measurement.

\section{Software and Data}
The implementation uses C++17, OpenMPI \cite{openmpi2026,mpiforum2023}, Python helper scripts, and a pretrained Ultralytics YOLO detector \cite{ultralytics2026}. The main dataset source is MOT17-derived video data \cite{motchallenge}. Compact MOT17 subsets are used for quick correctness and granularity experiments; longer sequence segments are used for the required input-size and speedup runs.

\section{Metrics}
The report uses the following metrics from the project reports and generated CSV design:
\begin{itemize}
    \item serial-versus-MPI correctness: mismatched frames and count error;
    \item detector accuracy: MAE, RMSE, percentage error, exact-match ratio;
    \item runtime with communication: wall-clock time including coordination;
    \item runtime without communication: compute-oriented time reported by the scripts;
    \item speedup and efficiency: \(S_p = T_1/T_p\), \(E_p = S_p/p\);
    \item idle-gap indicator for load-balance analysis.
\end{itemize}
